# 작업보고서 2026-03-03

## 작업 내용: 방문자 접속 로그 기능 구현

---

## 개요

블로그 방문자의 유입 경로(레퍼러), 검색 키워드, IP, 기기 유형 등을 수집하고 어드민에서 통계를 확인하는 기능 추가.

---

## 수집 데이터

| 필드 | 설명 |
|------|------|
| `page_type` | 페이지 종류 (home, review, recipe, news 등) |
| `page_slug` | 페이지 슬러그 |
| `page_url` | 방문 경로 |
| `referrer` | 유입 전체 URL |
| `referrer_domain` | 유입 도메인 (google.com, naver.com, direct 등) |
| `keyword` | 검색 키워드 (Google `q=`, Naver `query=`, Daum `q=` 파싱) |
| `utm_source/medium/campaign` | UTM 파라미터 |
| `ip_address` | 클라이언트 IP (Next.js에서 추출 후 전달) |
| `device_type` | 기기 유형 (mobile, tablet, desktop) |
| `created_at` | 방문 시간 |

## 변경 파일 목록

### 수정 (5개)

#### `automation-server/db/schema.sql`
- `visitor_logs` 테이블 추가 (`CREATE TABLE IF NOT EXISTS`)
- 인덱스 3개: `created_at`, `page_type`, `referrer_domain`
- 서버 재시작 시 `initializeSchema()`가 자동 실행 → 테이블 자동 생성

#### `automation-server/src/index.js`
- `POST /api/track` 공개 엔드포인트 추가
- 인증 미들웨어(authenticateToken) 전에 등록 → 인증 불필요
- Bot(`bot|crawler|spider`) 및 localhost(`127.0.0.1`, `::1`) 필터링
- `X-Forwarded-For` 헤더에서 IP 추출

#### `automation-server/src/routes/admin.js`
- `GET /api/admin/analytics/visitors` - 방문자 로그 목록 (페이징, 날짜/페이지타입/도메인 필터)
- `GET /api/admin/analytics/stats` - 집계 통계
  - 총 방문자 수
  - 날짜별 방문 수 (최근 30일)
  - 유입 소스 Top 10
  - 검색 키워드 Top 20
  - 기기 유형 분포
  - 페이지 유형 분포
  - 최근 방문 로그 20개

#### `web/app/layout.tsx`
- `<VisitorTracker />` 컴포넌트 임포트 및 `<Providers>` 내부에 삽입
- 어드민 경로(`/admin`)는 VisitorTracker 내부 로직에서 자동 제외

#### `web/components/admin/DashboardNav.tsx`
- "로그 뷰어" 옆에 "방문자 분석" 링크 추가 (`/admin/analytics`)

## 주의사항

- `POST /api/track`은 인증 없이 접근 가능 (publicRoutes)
- IP는 Docker Nginx 환경에서 `X-Forwarded-For`로 전달되므로 Next.js에서 추출 후 전달
- 국가 감지는 미구현 (추후 추가 예정)
- Bot(`bot|crawler|spider|crawling`) UA는 저장 안 함
- localhost/127.0.0.1 방문은 저장 안 함

## 후속 수정 (2026-03-03)

### 1. 어드민 UI 한글화 (`analytics/page.tsx`)
- 페이지 유형 한글 라벨 추가: `review` → 리뷰, `home` → 홈, `recipe` → 레시피, `news` → 뉴스 등
- 기기 유형 한글 라벨 추가: `mobile` → 모바일, `desktop` → 데스크탑, `tablet` → 태블릿
- 최근 방문 로그 테이블: 페이지 컬럼에 한글 라벨 + 슬러그 함께 표시

### 2. URL 인코딩 슬러그 디코딩
- **문제**: `%EC%BF%A0%ED%8...` 형태로 표시됨
- **원인**: 한글 슬러그가 URL 인코딩된 상태로 저장
- **수정**:
  - `VisitorTracker.tsx` `getPageSlug()` → `decodeURIComponent()` 적용 후 저장
  - `analytics/page.tsx` 테이블 표시 시에도 `decodeURIComponent()` 적용 (기존 데이터 대응)

### 3. 검색 키워드 섹션 개선
- 키워드 없을 때 빈 화면 대신 안내 문구 표시
- "네이버 검색만 수집 가능 (구글은 보안 정책상 미지원)" 설명 추가
- **Google 키워드 미수집 이유**: Google은 HTTPS Referrer-Policy 정책으로 검색 쿼리를 referrer에서 제거함. 코드 버그 아님, 브라우저/Google 정책 제한
- Naver(`?query=`), Daum(`?q=`), Bing(`?q=`)은 정상 수집
