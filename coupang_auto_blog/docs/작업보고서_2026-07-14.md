## 작업 1: 리뷰 페이지 쿠팡 CTA 확대 (클릭률 개선)

### 문제
- 쿠팡 버튼이 본문 맨 아래 1개뿐 → 스크롤을 끝까지 내린 방문자만 클릭 가능

### 수정 내용 (`web/components/ReviewPost.tsx`)
- **상단 대형 CTA**: 제목 바로 아래 "쿠팡 최저가 바로 확인하기" 버튼 + 가격 표시 (첫 화면 노출)
- **상품 이미지 클릭**: 썸네일 클릭 시 쿠팡 새 탭 ("이미지 클릭 → 쿠팡에서 보기" 안내 배지로 명시)
- **본문 중간 CTA**: 문단 8개 이상 긴 글에 자동 삽입
- **하단 고정(스티키) 바**: 300px 스크롤 후 화면 하단에 상시 노출 (모바일 효과 큼)
- 기존 하단 상품 카드에 가격 표시 추가
- slug 없는 구형 `/review/[id]` 페이지에도 상단 CTA 추가

## 작업 2: AI 리뷰 SEO 강화 (글 내용·메타·URL)

### 문제
- 리뷰 분량 400~600자는 검색엔진이 '얇은 콘텐츠'로 취급 → 노출 불리
- 메타 설명이 150자에서 문장 중간에 뚝 잘림
- `/api/review/publish`가 `review-{id}-{timestamp}` 형태의 키워드 없는 slug 생성
- 재발행 시 slug가 바뀌어 기존 검색 유입이 끊기는 문제

### 수정 내용
- **권장 프롬프트 개선** (`web/types/settings.ts`, `automation-server/src/services/settingsService.js`)
  - 800~1,200자, 도입→첫인상→사용경험→장단점→꿀팁→추천대상 구조
  - 검색 노출 규칙 추가 (상품명 2~3회 자연 포함, 검색성 표현 사용)
- 어드민 "권장 프롬프트 적용" 버튼이 글자수(800~1200)도 함께 적용하도록 변경
- **메타 설명**: 문장 경계에서 자르는 `buildMetaDescription()` 적용
- **SEO 제목**: `{정제된 상품명} 솔직 후기 · 쿠팡 최저가 | 세모링크` (롱테일 검색 대응)
- **발행 slug**: 상품명 기반 한글 slug 생성, 기존 slug는 보존
- 서버 DB의 기본 템플릿을 SQL로 직접 권장 버전으로 갱신 완료 (어드민 조작 불필요)

### 검증 결과 (서버)
- 테스트 생성: 리뷰 #365, **1,093자** (목표 범위 적중), toneScore 0.64
- 발행 slug: `차일드라이프-멀티비타민-미네랄-망고-오렌지맛-1개` (상품명 기반)
- SEO 제목·메타 설명 정상 확인, 페이지 200 응답

### 커밋
- `af6d14e` feat: 리뷰 SEO 강화 - 프롬프트 개선, 메타설명 문장 자르기, 상품명 slug

## 작업 3: 쿠팡 링크 클릭 추적 시스템

### 내용
- **DB**: `coupang_clicks` 테이블 신설 (`automation-server/db/schema.sql`)
  - review_id, review_slug, product_name, position, referrer_domain, device_type, ip 등
- **수집**: `POST /api/track/click` 공개 엔드포인트 (봇·내부 IP 필터)
  - Next.js 프록시 `web/app/api/track/click/route.ts` (IP·기기·유입 도메인 추출)
- **프론트**: 리뷰 페이지 5개 CTA에 위치 구분 클릭 이벤트 부착
  - `top`(상단) / `image`(이미지) / `mid`(중간) / `bottom`(하단 카드) / `sticky`(고정바)
- **통계 API**: `GET /api/admin/analytics/clicks` (기간별 총 클릭, 위치별, 날짜별, 글별 Top20 + CTR)
- **어드민 UI**: 방문자 분석 페이지에 추가
  - 핵심 지표 카드 3종: 총 방문자 / 쿠팡 링크 클릭 / 리뷰→쿠팡 클릭률
  - 버튼 위치별 클릭, 글별 클릭 Top 20 표 (클릭/조회/CTR)

### 검증 결과 (서버)
