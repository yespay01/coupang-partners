## 2. Dockerfile 빌드 캐시 최적화

**문제**: `COPY . .` 한 줄이 소스코드와 `public/` 폴더를 같은 레이어에 묶어, `public/`에 파일 하나만 추가해도 이후 모든 레이어 캐시가 무효화됨

**수정**: BuildKit 캐시 마운트 적용

```dockerfile
# 이전
RUN npm run build

# 이후
RUN --mount=type=cache,target=/app/.next/cache \
    npm run build
```

**효과**: `public/`만 변경된 경우 Next.js 증분 빌드 가능 → 빌드 시간 대폭 단축

> 이 변경 적용 시 서버에서 최초 1회 `--no-cache` 빌드 필요

## 5. 사이트맵 동적 URL 누락 수정

**문제**: 빌드 후 `https://semolink.store/sitemap.xml`에 리뷰/레시피/뉴스 URL이 사라짐

**원인 분석**:
- `sitemap.ts`의 fetch에 `next: { revalidate: 3600 }` 옵션이 있었음
- `docker compose build` 시 Next.js가 sitemap을 사전 렌더링함
- 이 시점에 automation-server가 아직 미기동 상태 → fetch 실패
- 실패 결과가 3600초 동안 Next.js fetch 캐시에 저장
- 배포 후 1시간 동안 정적 페이지만 나오는 현상 발생

**수정**: 3개 fetch 모두 `cache: "no-store"` 로 변경

```typescript
// 이전
const res = await fetch(`${AUTOMATION_SERVER_URL}/api/reviews?limit=1000`, {
  next: { revalidate: 3600 },
});

// 이후
const res = await fetch(`${AUTOMATION_SERVER_URL}/api/reviews?limit=1000`, {
  cache: "no-store",
});
```

**효과**: 사이트맵 재생성 시 항상 최신 데이터 반영 (페이지 레벨 `revalidate: 3600`은 유지)
