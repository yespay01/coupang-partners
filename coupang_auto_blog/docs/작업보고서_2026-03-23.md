## 배경

구글 서치콘솔에서 다음 색인 문제 확인:
- **발견됨 - 현재 색인이 생성되지 않음**: `/reviews/*`, `/recipes/*`, `/news/*`, `/search` 전체
- **크롤링됨 - 현재 색인이 생성되지 않음**: 최근 리뷰 페이지들 (3/22 크롤링)
- **사용자가 선택한 표준이 없는 중복 페이지**: `/review/29` (구 경로)
- **찾을 수 없음(404)**: woff 폰트 파일

---

## 원인 분석

### 핵심 원인: 목록 페이지 전체 `"use client"`

| 페이지 | 문제 |
|--------|------|
| `app/page.tsx` (홈) | `"use client"` → 구글봇 크롤 시 빈 HTML만 반환 |
| `app/reviews/page.tsx` | `"use client"` → 동일 |
| `app/news/page.tsx` | `"use client"` → 동일 |
| `app/recipes/page.tsx` | `"use client"` → 동일 |
| `app/review/[id]/page.tsx` | `"use client"` + canonical 없음 → 중복 페이지 |

**상세 페이지** (`reviews/[slug]`, `news/[id]`, `recipes/[id]`)는 이미 SSR로 구현되어 있어 문제 없음.

### 과거 SEO 작업 현황
- `docs/SEO_OPTIMIZATION_2026-02-19.md`는 **다른 프로젝트(yespos.co.kr)** 문서로 semolink.store와 무관
- semolink.store에는 sitemap, robots.txt, 상세 페이지 metadata는 구현되어 있었으나 **목록 페이지 SSR 전환은 누락**

## 문제 1: 빌드 중 Retrying 1/3 메시지 수백 개

### 원인
Server Component의 fetch 함수들이 빌드 중 automation-server에 접속 시도 → 미응답 → retry

### 해결: NEXT_PHASE 조건 추가

빌드 단계(`phase-production-build`)에서는 fetch를 건너뛰고 즉시 빈 값 반환.

**수정 파일 (총 5개):**

| 파일 | 함수 | 반환값 |
|------|------|--------|
| `web/app/page.tsx` | `fetchPublishedReviews` | `[]` |
| `web/app/news/page.tsx` | `fetchInitialNews` | `{ news: [], hasMore: false }` |
| `web/app/recipes/page.tsx` | `fetchInitialRecipes` | `{ recipes: [], hasMore: false }` |
| `web/app/reviews/page.tsx` | `fetchInitialReviews` | `{ reviews: [], hasMore: false }` |
| `web/app/sitemap.ts` | `sitemap` | `[]` |

```typescript
// 추가된 한 줄 (각 fetch 함수 맨 위)
if (process.env.NEXT_PHASE === 'phase-production-build') return [];
```

### sitemap 부수 효과
빌드 중 fetch가 없어지면서 sitemap이 **ƒ Dynamic → ○ Static (1h revalidate)** 으로 변경됨.
런타임에서는 `revalidate = 3600`으로 1시간마다 실제 데이터로 재생성.

## 문제 2: npm 취약점 (fast-xml-parser, minio)

### 원인
`minio` 패키지가 취약한 버전의 `fast-xml-parser`에 의존

### 해결
```bash
cd automation-server && npm install minio@latest
```
→ `minio 8.0.7`, `fast-xml-parser 5.5.8`으로 업그레이드 → 취약점 0개

### package-lock.json 이슈
`.gitignore`에 `package-lock.json` 전역 제외 → 서버 빌드 시 lock 불일치 오류 발생
→ `-f` 플래그로 강제 추가:
```bash
git add -f automation-server/package-lock.json
```
