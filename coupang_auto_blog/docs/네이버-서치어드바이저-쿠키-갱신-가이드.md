# 네이버 서치어드바이저 쿠키 갱신 가이드

## 언제 갱신이 필요한가?

- 어드민 분석 페이지(`/admin/analytics`)에서 **"쿠키 만료"** 배지가 표시될 때
- Slack으로 **"네이버 서치어드바이저 세션 쿠키가 만료되었습니다"** 알림을 받았을 때
- 네이버 키워드 섹션에 **"쿠키 갱신 필요"** 메시지가 나올 때

> 평상시에는 서버가 6시간마다 자동으로 세션을 유지하므로 갱신이 필요 없습니다.

---

## 갱신 방법

### 방법 1: Python 스크립트 사용 (권장)

#### 1단계: 크롬에서 네이버 서치어드바이저 접속
1. 크롬 브라우저에서 https://searchadvisor.naver.com 접속
2. 네이버 계정으로 로그인
3. 사이트 목록에서 `semolink.store` 클릭

#### 2단계: 쿠키 값 복사
1. `F12` 키를 눌러 개발자 도구 열기
2. **Network** 탭 클릭
3. 페이지를 새로고침 (F5)
4. 요청 목록에서 아무 항목 하나 클릭
5. 오른쪽 패널에서 **Headers** 탭 → **Request Headers** 섹션
6. `Cookie:` 항목의 값 전체를 복사 (우클릭 → Copy value)

#### 3단계: 스크립트 실행
```bash
cd C:\Users\sakai\OneDrive\바탕 화면\Coupang partnner\coupang_auto_blog
python tools\naver-cookie-sync.py
```

- 자동 읽기가 실패하면 수동 입력 모드로 전환됨
- `Cookie 값:` 프롬프트에 복사한 쿠키를 붙여넣기
- `admin_session 값:` 프롬프트에 어드민 세션 쿠키 입력

#### admin_session 값 확인 방법
1. 크롬에서 https://semolink.store/admin 접속 (로그인 상태)
2. `F12` → **Application** 탭 → 왼쪽 **Cookies** → `https://semolink.store`
3. `admin_session` 항목의 **Value** 복사

### 방법 2: 서버에서 직접 설정

SSH로 서버에 접속하여 직접 DB에 저장하는 방법입니다.

```bash
ssh insuk@192.168.0.5

# 쿠키 문자열을 변수에 저장 (아래 값을 실제 쿠키로 교체)
COOKIES='NID_AUT=...; NID_SES=...; (전체 쿠키 문자열)'

# Docker 컨테이너에서 직접 실행
docker exec coupang-automation node -e "
const {saveNaverSaCookies} = require('./src/services/naverSearchAdvisor.js');
saveNaverSaCookies('$COOKIES').then(() => {
  console.log('저장 완료');
  process.exit(0);
});
"
```

## 참고 사항

### 시스템 구조
```
크롬(네이버 로그인) → 쿠키 복사 → Python 스크립트 → 서버 API → PostgreSQL DB 저장
                                                                    ↓
                                        서버 cron (6시간마다) ← 네이버 SA API 호출
                                              ↓
                                        세션 만료 시 Slack 알림
```

### 관련 파일
| 파일 | 역할 |
|------|------|
| `tools/naver-cookie-sync.py` | 쿠키 동기화 스크립트 |
| `automation-server/src/services/naverSearchAdvisor.js` | 네이버 SA API 호출 + 세션 갱신 |
| `automation-server/src/cron/scheduler.js` | 6시간 keep-alive cron |
| `automation-server/src/routes/admin.js` | 쿠키 업데이트 API 엔드포인트 |

### 환경변수 (서버 automation-server/.env)
| 변수 | 용도 |
|------|------|
| `NAVER_SA_COOKIES` | 초기 쿠키값 (DB 미설정 시 fallback) |
| `NAVER_SA_SITE_HASH` | 사이트 해시 (`4b2cc44a...`) |

> DB에 쿠키가 저장되면 환경변수보다 DB 값이 우선 사용됩니다.
