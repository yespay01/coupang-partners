# 작업보고서 2026-03-23

## 작업 내용: 구글 서치콘솔 색인 문제 해결 (SSR 전환)

---

## 배경

구글 서치콘솔에서 다음 색인 문제 확인:
- **발견됨 - 현재 색인이 생성되지 않음**: `/reviews/*`, `/recipes/*`, `/news/*`, `/search` 전체
- **크롤링됨 - 현재 색인이 생성되지 않음**: 최근 리뷰 페이지들 (3/22 크롤링)
- **사용자가 선택한 표준이 없는 중복 페이지**: `/review/29` (구 경로)
- **찾을 수 없음(404)**: woff 폰트 파일

---

## 원인 분석

### 핵심 원인: 목록 페이지 전체 `"use client"`

| 페이지 | 문제 |
|--------|------|
| `app/page.tsx` (홈) | `"use client"` → 구글봇 크롤 시 빈 HTML만 반환 |
| `app/reviews/page.tsx` | `"use client"` → 동일 |
| `app/news/page.tsx` | `"use client"` → 동일 |
| `app/recipes/page.tsx` | `"use client"` → 동일 |
| `app/review/[id]/page.tsx` | `"use client"` + canonical 없음 → 중복 페이지 |

**상세 페이지** (`reviews/[slug]`, `news/[id]`, `recipes/[id]`)는 이미 SSR로 구현되어 있어 문제 없음.

### 과거 SEO 작업 현황
- `docs/SEO_OPTIMIZATION_2026-02-19.md`는 **다른 프로젝트(yespos.co.kr)** 문서로 semolink.store와 무관
- semolink.store에는 sitemap, robots.txt, 상세 페이지 metadata는 구현되어 있었으나 **목록 페이지 SSR 전환은 누락**

## 수정 내용

### 수정된 파일 (5개)

#### `web/app/page.tsx` (홈)
- `"use client"` 제거
- `async` Server Component로 전환
- `AUTOMATION_SERVER_URL`에서 직접 fetch (`revalidate: 3600`)
- metadata + canonical 추가

```typescript
export const metadata: Metadata = {
  title: "세모링크 - 세상의 모든 링크",
  description: "세상의 모든 링크가 모이는 허브...",
  alternates: { canonical: "https://semolink.store" },
};
export const revalidate = 3600;
```

#### `web/app/reviews/page.tsx`
- `"use client"` 제거 → Server Component
- metadata + canonical 추가
- 서버에서 초기 12개 리뷰 fetch
- 인터랙션(카테고리 필터, 더보기)은 `ReviewsListClient`로 분리
- `useSearchParams` 대신 Server Component의 `searchParams` prop 사용

#### `web/app/news/page.tsx`
- `"use client"` 제거 → Server Component
- metadata + canonical 추가
- 서버에서 초기 12개 뉴스 fetch
- 더보기 버튼은 `NewsListClient`로 분리

#### `web/app/recipes/page.tsx`
- `"use client"` 제거 → Server Component
- metadata + canonical 추가
- 서버에서 초기 12개 레시피 fetch
- 더보기 버튼은 `RecipesListClient`로 분리

#### `web/app/review/[id]/page.tsx` (구 경로)
- `"use client"` 제거 → Server Component
- slug가 있으면 `permanentRedirect('/reviews/[slug]')` → 구글 중복 페이지 문제 해결
- slug가 없는 경우 canonical 태그 추가

### 신규 생성 파일 (3개)

#### `web/components/ReviewsListClient.tsx`
- `"use client"` 컴포넌트
- props: `initialReviews`, `initialHasMore`
- 카테고리 필터 (클라이언트 사이드)
- 더보기 버튼 (`/api/reviews` fetch)

#### `web/components/NewsListClient.tsx`
- `"use client"` 컴포넌트
- props: `initialNews`, `initialHasMore`
- 더보기 버튼 (`/api/news` fetch)

#### `web/components/RecipesListClient.tsx`
- `"use client"` 컴포넌트
- props: `initialRecipes`, `initialHasMore`
- 더보기 버튼 (`/api/recipes` fetch)

## 아키텍처 변경 (목록 페이지 패턴)

```
이전:
page.tsx ("use client")
└── useEffect → fetch → render (구글봇: 빈 HTML)

이후:
page.tsx (Server Component, async)
├── export metadata (title, description, canonical)
├── fetch 초기 데이터 from AUTOMATION_SERVER_URL (서버에서 실행)
└── *ListClient.tsx ("use client")
    ├── useState로 추가 데이터 관리
    └── 더보기/필터 등 인터랙션 처리
```
