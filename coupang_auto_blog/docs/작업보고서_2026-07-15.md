## 작업 3: 네이버 SA 쿠키 자동 수집 + 세션 유지 시스템

### 문제
- 네이버 서치어드바이저는 공식 API가 없어 세션 쿠키 인증 필요
- 쿠키가 만료되면 수동 갱신해야 하는 불편

### 수정 내용
- **DB 저장**: PostgreSQL `settings` 테이블에 쿠키 저장 (환경변수 fallback)
  - `INSERT ... ON CONFLICT (key) DO UPDATE` 패턴
- **세션 유지 cron**: 6시간마다 `refreshNaverSession()` 실행 (`scheduler.js`)
  - 서치어드바이저 콘솔 페이지 방문으로 세션 갱신
  - 302 → 로그인 리다이렉트 감지 시 Slack 알림 발송
- **쿠키 업데이트 API**: `PUT /api/admin/credentials/naver-sa` (인증 필요)
- **상태 확인 API**: `GET /api/admin/credentials/naver-sa/status`
- **Python 스크립트**: `tools/naver-cookie-sync.py`
  - 자동 읽기(browser-cookie3) 시도 → 실패 시 수동 붙여넣기 모드
  - admin_session 쿠키로 인증하여 서버 API에 전송
- **운영 문서**: `docs/네이버-서치어드바이저-쿠키-갱신-가이드.md` 작성

### 시스템 구조
```
[쿠키 갱신 시] 크롬 로그인 → 쿠키 복사 → Python 스크립트 → 서버 API → DB 저장
[평상시]       서버 cron (6시간마다) → 네이버 SA 페이지 방문 → 세션 유지
[만료 감지]    302 리다이렉트 → Slack 알림 → 어드민 페이지 "쿠키 만료" 배지
```

## 작업 4: 방문자 분석 페이지 UI 대시보드 스타일 개편

### 문제
- 10개 섹션이 세로로 나열되어 페이지가 길고 정보 파악이 어려움

### 수정 내용 (`web/app/(dashboard)/admin/analytics/page.tsx`)
- **KPI 카드**: 5개 핵심 지표를 가로 한 줄 배치
  - 방문자 / 쿠팡 클릭 / 클릭률 / 구글 노출 / 네이버 노출
- **날짜 필터**: 세그먼트 pill 스타일로 변경 (7일/14일/30일/90일)
- **2컬럼 그리드 레이아웃**:
  - 날짜별 방문 추이 | 유입 소스 Top 10
  - 기기 유형 | 페이지 유형별 방문
- **검색 키워드 탭 UI**: 구글/네이버 키워드를 하나의 섹션으로 통합, 탭 전환
- **최근 방문 로그**: 기본 접혀있고 "펼치기" 버튼으로 토글
- **스타일 통일**: `rounded-xl bg-white shadow-sm border border-slate-200` 카드, striped 테이블
