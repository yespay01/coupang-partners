## 문서 작성 가이드

새로운 문서를 추가할 때는 다음 규칙을 따라주세요:

### 문서 종류별 저장 위치

| 문서 종류 | 저장 위치 | 예시 |
|---------|---------|-----|
| 프로젝트 기획서 | `docs/planning/` | 기획서_v1.0.md |
| 개발 계획서 | `docs/planning/` | 개발계획서_v2.0.md |
| 기술 스펙 | `docs/specs/` | API_명세서.md |
| 개발 가이드 | `docs/` | 배포_가이드.md |
| 참고 자료 | `docs/references/` | 시장조사_보고서.pdf |
| 구버전 문서 | `docs/archive/` | 기획서_v1.0.md |
| 미디어 파일 | `resources/media/` | 강의_녹음.m4a |

### 파일 네이밍 규칙

- **한글 문서**: `문서명_v버전.확장자` (예: `기획서_v3.0.md`)
- **영문 문서**: `document_name_v1.0.ext` (예: `api_spec_v1.0.md`)
- **버전 표기**: 주요 변경 시 정수 증가 (v1.0 → v2.0), 소소한 수정 시 소수점 증가 (v1.0 → v1.1)

### 문서 업데이트 시

1. **기존 문서 보존**: 구버전은 `docs/archive/`로 이동
2. **버전 업데이트**: 파일명에 새 버전 번호 반영
3. **인덱스 업데이트**: 이 README.md 파일 업데이트
4. **링크 확인**: 다른 문서의 링크가 깨지지 않았는지 확인
